Dataset Name:
structural_steel_beam_midspan_deflection_dataset_imperial

Problem Statement:
This dataset is the imperial-unit version of a synthetic but engineering-grounded tabular dataset for teaching regression machine learning in Structural Engineering. The task is to predict the maximum midspan deflection of simply supported steel I-beams under service loading using realistic geometric, material, and loading variables expressed in US customary units.

Chosen Theme:
Predicting maximum midspan deflection of simply supported steel I-beams under combined distributed and concentrated service loads.

Dataset Size:
- Rows: 1000
- Input features: 14
- Output variables: 1
- Machine learning task: Regression

Target Variable:
- Name: Maximum Midspan Deflection (in)
- Description: Net downward serviceability deflection at beam midspan after accounting for combined service loading, composite stiffness effects, partial camber offset, and moderate modeling and construction variability.
- Unit: in
- Observed range in this dataset: 0.025 to 2.806

How the Synthetic Data Was Constructed:
1. Structural context:
   The records represent simply supported steel I-beams used as floor or framing beams in buildings. Inputs were generated within realistic ranges for span, tributary width, service floor loads, and section geometry.

2. Geometry generation:
   Beam depth, flange width, flange thickness, and web thickness were generated as internally consistent I-section dimensions using realistic proportion rules for compact to moderately slender rolled or built-up steel beams. Geometry was not sampled independently without checks. Section dimensions were correlated with span and service demand so that deeper sections tend to occur in longer or more heavily loaded beams.

3. Material properties:
   Steel yield strength was sampled around common structural steel grades from about 35 ksi to 63 ksi with small variability. Elastic modulus was centered near 29,000 ksi with modest variation to reflect standard structural steel behavior.

4. Loading logic:
   Area loads were represented by superimposed dead load and live load in psf, then converted to line load through tributary width. Beam self-weight was calculated from the generated cross-sectional area using a structural steel unit weight of about 490 pcf. An additional concentrated service load was included for cases with equipment or localized use. Point load position was allowed to vary along the span.

5. Structural response model:
   The target deflection was computed from classical elastic beam theory:
   - Uniform load component based on delta = 5wL^4 / (384EI) for a simply supported beam
   - Concentrated load component based on the closed-form elastic solution for a point load at an arbitrary location
   Major-axis second moment of area was calculated from the synthetic I-section dimensions. Composite action ratio increased effective flexural stiffness. Initial camber reduced the net measured downward deflection. Moderate noise was then added to represent connection flexibility, erection tolerance, and modeling uncertainty.

6. Plausibility checks:
   Candidate rows were rejected if they produced unrealistic section proportions, implausible self-weight, excessively low or high service stress utilization, or extreme deflection values not consistent with common building beam behavior. This avoids impossible or contradictory combinations.

Engineering Logic Behind the Relationships:
- Deflection increases strongly with span length, especially because elastic beam deflection scales approximately with L^4 under uniform load.
- Deflection decreases with greater flexural stiffness EI, which is driven mainly by section geometry and elastic modulus.
- Tributary width converts area load to beam line load, so larger tributary width generally increases deflection.
- Superimposed dead load, live load, and additional point load increase deflection.
- Composite action increases effective stiffness and therefore tends to reduce deflection.
- Initial camber offsets part of the service deflection.
- Yield strength has only a secondary influence on serviceability deflection, which is realistic and useful for beginner feature importance studies.
- Because real beams are selected in response to span and load demands, larger sections can still appear together with larger deflections. This introduces realistic confounding rather than an artificially simple inverse relationship.

Missing Data Strategy:
Each input feature column contains randomly distributed missing values represented as NaN. Missingness was injected after the target was calculated from the complete underlying engineering variables. This mimics incomplete field records, incomplete drawings, or missing material and load metadata while preserving a fully observed regression target.

Intended Educational Use:
This dataset is suitable for beginner graduate students learning:
- data cleaning and missing value imputation
- exploratory data analysis and visualization
- correlation analysis and engineering interpretation
- baseline linear regression
- nonlinear regression and feature engineering
- train/test splitting and cross-validation
- model comparison and error metrics
- discussion of confounding, multicollinearity, and physically informed feature importance

Assumptions:
- Beams are simply supported and prismatic.
- Response is governed by elastic serviceability behavior in major-axis bending.
- Loads are service-level loads, not factored ultimate design combinations.
- Composite action is represented by a single stiffness amplification proxy rather than an explicit slab model.
- Camber only partially offsets the final net deflection.
- Local buckling, lateral-torsional buckling, vibration, temperature effects, long-term creep, and connection details are not modeled explicitly.
- Hidden occupancy and use classes were used internally to diversify load patterns, but those class labels are not included in the final dataset so that the table remains beginner-friendly and purely numeric.
